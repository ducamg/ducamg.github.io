使用

  • 查询

    1. 查询单个参数

      • 基本类型: 取值:#{参数名}

    2. 查询多个参数

      • 取值: #{参数名} 无效

      • 可用: 0,1(参数索引)或者arg0,arg1,arg2

      • 原因:只要传入多个参数，mybatis会将这些参数封装在map中，封装时使用的key就是参数的索引和参数的第几个标识

      • #{key} 就是从map中取值

        1. 封装自己指定的key 加入param注解 :命名参数

    3. 传入map

      • #{key}

    4. 传入POJO

      • 取值:#{pojo.属性名}

    5. 扩展 多个参数自动封装map

      1. 取值的时候可以设置一些规则:id=#{id,jdbcType=INTEGER}
        • javaType
        • jdbcType
        • mode
        • numericScale
        • resultMap
        • typeHandler
        • jdbcTypeName
        • expression
        • 只有jdbcType才可能使用的，默认不指定也行，但是传入的数据是null
          • MySQL没问题
          • oracle就会报错
      • mybatis有两种取值方式
        1. #{属性名}:是预编译的方式，参数的位置都是用问好的方式替代,参数都是预编译设置进去的:安全，不会有sql注入
          • id=#{id} and empname =#{empname}

            Preparing: select * from t_employee where id=? and empname =?
        2. ${属性名}:不是参数预编译，直接和sql语句进行拼串：不安全
          使用场景:一般使用#{}安全,在不支持预编译的位置要取值就使用${}
          • 获取表名
          • id=${id} and empname =#{empname}

            select * from t_employee where id=1 and empname =?

驼峰命名法

  • 默认mybatis 自动封装结果集
    1. 按照列名和属性名一一对应规则(不区分大小写)(数据库:aaa javaBean:aaa)
    2. 如果不一一对应:
      1. 开启驼峰命名方法(满足驼峰命名规则:数据库:a_bbb javaBean:aBbb )
      2. 起别名
      3. 自定义结果集:自己定义每一列数据和javabean的映射规则
