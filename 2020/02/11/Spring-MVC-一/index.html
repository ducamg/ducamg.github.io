简介

SpringMVC=Spring的web模块和MVC:新的软件架构模式

  • M(Model模型)：封装和映射数据(javabean)

    • 业务模型
    • 数据模型

  • V(View视图):界面显示(.jsp / .html)

  • C(Controller控制器)：整个网站的跳转逻辑(servlet)

MVC:每一层只编写自己的东西。不写其他代码

控制器：调用业务逻辑处理，跳转到某个页面

  • 分层是为了解耦，解耦为了和维护方便和分工合作

使用

导入web模块

  • spring-webmvc-5.2.8.RELEASE.jar
  • spring-web-5.2.8.RELEASE.jar

配置

  1. web.xml

  2. dispatcher-servlet.xml

测试

  1. 创建controller文件

细节

运行流程

  • 客户端点击链接会发送http://localhost:8080/4_SpringMVC_02/hello 请求
  • 来到Tomcat服务器
  • SpringMVC前端控制器收到所有请求
  • 来请求地址和@RequestMapping标注的哪个匹配，来找到哪个类的哪个方法来处理
  • 前端控制器找到目标处理类和目标方法，直接利用反射执行目标方法
  • 方法执行完成会有一个返回值：SpringMVC认为这个返回值就是要去页面地址
  • 获取到返回值后，用视图解析器进行拼串得到完整页面地址
  • 拿到页面地址，前端控制器转发到页面

@RequestMapping

  • 就是用来告诉SpringMVC：这个方法用来处理什么请求

    • /hello的/最好不要省略(是可以省略的)，即使省略的也是从当前项目下开始,习惯加上

不指定配置文件位置

  • 如果不指定也会去默认找一个文件： /WEB-INF/springDispatcherServlet-servlet.xml
  • 可以在/WEB-INF/ 下创建一个名为： 前端控制器-servlet.xml

<url-pattern>

  1. /和 /* 的区别

    • Tomcat服务的web.xml文件中有个DefultServlet的url-patttern=/

      前端控制器中配置的url-pattern=/

      • 所有的项目的web.xml都是继承于tomcat的web.xml的

      • DefultServlet是Tomcat中处理静态资源的

        • 除过jsp和servlet文件外剩下的都是静态资源

          index.html是静态资源 Tomcat就会在服务器下找到这个资源并返回

          相当于前端控制器覆盖了Tomcat服务器中DefultServlet

          • 静态资源会来到DispatcherServlet(前端控制器)，查看哪个方法的RequestMapping是index.html

        • 为什么jsp有能访问？因为我们没有覆盖服务器中的JspServlet的配置

      • /* 直接是拦截所有请求 : 写/也是为了迎合Rest风格的URL地址
