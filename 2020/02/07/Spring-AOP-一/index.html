AOP:(Aspect Oriented Programming) 面向切面编程

  • 指基于OOP(面向对象编程)基础之上的新的编程思想

面向切面编程:

将某段代码（日志代码）动态切入（不把日志代码写到业务逻辑中去）到指定方法（加减乘除）的指定位置（方法的开始结束异常）


spring简化了面向切面编程。

AOP专业术语

  • 切面类
  • 横切关注点
  • 通知方法
  • 连接点
  • 切入点
  • 切入点表达式

Spring 切面类里面的方法什么时候运行

  1. @Before 方法之前运行 　 :前置通知
  2. @After 方法运行结束之后运行 　　　　 :后置通知
  3. @AfterReturning 方法正常返回之后运行　 :返回通知
  4. @AfterThrowing 方法抛出异常之后运行　　 :异常通知
  5. @Around 环绕　　　　　　　 :环绕通知

切入点表达式

语法：execution(访问权限符　返回值类型　方法签名(参数表))

  • 通配符

    1. \*
      • 匹配一个或者多个字符:
        @Before(“execution(public int ducam.gold.impl.My*.*(int,int)”
      • 匹配任意一个参数:
        @Before(“execution(public int ducam.gold.impl.My*.(int,\)”
      • 只能匹配一层路径:
        @Before(“execution(public int ducam.gold.impl..(int,*)
      • 权限位置不写(public
    2. ..
      • 任意匹配多个参数:
        @Before(“execution(public int ducam.gold.impl.My*.*(..)”
      • 任意匹配多层路径:
        @Before(“execution(public int ducam..My*.*(..)”
    • 模糊写法： execution(* 　*.\*(..)) (极不推荐)
    • 精确写法:
      @Before(“execution(public int ducam.gold.impl.MyCalculator.add(int,int))”)
    • 支持 && || !(与或非):
      • execution(public int ducam.gold..Cal*.*(..)) && execution(* 　.\(int,int)) 必须满足两种

  • 格式：@注解(“execution(方法全类名)”)

    • @Before(“execution(public int ducam.gold.impl.MyClaculator.add(int,int)”)
    • @Before(“execution(public int ducam.gold.impl.MyClaculator.*(int,int)”)

  • 开启基于注解的Aop(使用名称空间)

    • <aop:aspectj-autoproxy>

    • 强制使用cglib代理

通知方法的执行顺序

  1. 正常执行 @Before —>@After —> @AfterReturning
  2. 异常执行 @Before —>@After —->@AfterThrowing

@Around(环绕通知)

  1. 四合一就是环绕通知

  2. 环绕通知的参数

  3. 执行顺序

    环绕通知优先于普通通知执行

    环绕前置—普通前置—目标方法执行—环绕正常返回/异常—环绕后置—普遍后置—普通返回/异常
    注意：环绕通知接收到异常后要及时抛出

  4. 多切面运行顺序:环绕通知作用于谁就在谁的前面结束
